3.4.4 DEBUG Statement

The SQL statements are translated into MAL programs, which are optimized and stored away in an sql_cache module. The generated code can be debugged with the MAL debugger. It provides a simple mechanism to trace the execution, hunting for possible errors and detect performance bottlenecks (Runtime Inspection).

The example below, illustrates how you can easily obtain a quick overview of the cost components of the query using the debugger timer flag and continuation command for the debugger.

[ERROR: the 'start transaction' is needed now] 

>start transaction;
&4 f
>debug select count(*) from tables;
#mdb #    mdb.start();
mdb>next
#mdb #    sql_cache.s0_0();
mdb>timer
mdb>continue
