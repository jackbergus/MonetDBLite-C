MonetDB

Overview

The technical documentation is spread over a number of topics. The command line tools are Mserver and MapiClient, which can be used to establish a session.

The topics covered in Kernel are mostly detailed technical descriptions of the MonetDB kernel; APIs contains an overview of the available application programming interfaces, and Modules contains detailed descriptions of the database kernel extensions. The guides provides details on the SQL front-end and facilities handle XML in MonetDB.

(For more information visit the MonetDB website.)

Kernel

The technical documentation of MonetDB is mainly geared at application developers and those interested in the inner working of the system. The starting point for getting acquainted with the system is to read the architecture overview. Although it is rather technically oriented, it creates a general understanding of its design and inner working.

The User Guide of the Monet Interpreter Language (MIL) explains the algebraic interface of the Monet back-end. Front-ends like the SQL front-end translate high level query languages to MIL. This document is a must if you plan to develop your own front-end, or need low-level access to the MonetDB server. An overview of the architecture and features of the SQL front-end is described separately. Tutorials on SQL syntax and semantics are abundantly available on the web and thus not included here.

The Monet Extension Language (MEL) is a specification language for creating extension modules that add new algebra-operators and atomic types to MIL. Implementations should be provided separately in C/C++.
The utility that parses MEL specifications and generates glue code is also called mel.

Extension programmers are advised to first study the technical documentation of the very kernel to get acquainted with the most important internal data structures and APIs. More technical information about MonetDB is available on our website.

Much of this documentation is integrated with the MonetDB code using the literate programming tool Mx.

APIs

The application programmer's interfaces (APIs) have been developed with a keen eye on what currently prevails in the database community. The native client-server interface of MonetDB is called Mapi (MonetDB API). Internally, it is used to implement all other APIs. Mapi is language neutral, it can both be used for MIL and SQL querying.
There are language bindings for (among others) C and Java.

Check our website for more information on MonetDB's APIs.

Modules

Monet is an extensible system. It obtains its functionality from a large collection of extension modules. They have been developed over the years to cover our need in experiments. Construction of a module requires intimate knowledge of the internals of MonetDB. Creation of a new module should only be considered if the current set has a serious functional limitation, or if a particular application the performance requirements can not be met by SQL or MIL.

The modules can be dynamically loaded (and dropped) during a session, but often it makes more sense to load the module set as part of a session initialization action. Each client has its own set of modules.

The modules can be roughly grouped as follows. The algebra, bat, builtin contain the binary algebra primitives. The arith, mmath, decimal modules provide various forms of scalar processing. The user defined types supported are blob, decimal, enum, monettime. The primitives to support datamining applications, e.g. cross-table construction and advanced group-based statistics, are covered by aggr, aggrX3, xtables. Support for performance monitoring is included in counters, pcl, calib
